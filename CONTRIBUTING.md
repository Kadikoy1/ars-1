# Contributing to ARS-1

Thank you for your interest in ARS-1. The standard is published under CC0 and is open for free implementation. Public comment on v0.2 is welcome until **31 August 2026**.

## Ways to contribute

### Open an issue

Use [GitHub Issues](https://github.com/Kadikoy1/ars-1/issues) for:

- Technical review of the JSON schemas
- Feedback on the message lifecycle
- Issues with the ISO 20022 mapping
- Travel Rule integration questions
- Edge cases in conditionality, hardship, or reversal
- Use cases not currently covered by the three deployment profiles
- Typos or clarifications in the specification

Please tag issues with the relevant area (`schema`, `iso20022`, `travel-rule`, `profile-institutional`, `profile-commercial`, `profile-sovereign`, `spec-typo`, `use-case`).

### Submit a pull request

PRs are welcome for:

- JSON Schema refinements (with test vectors)
- Worked examples (with explanation)
- ISO 20022 mapping table corrections
- Translations of the README or profile documents
- Reference implementation sketches

For substantive changes to the specification itself, please open an issue first to discuss.

### Email or web form

Substantive feedback that doesn't fit a GitHub issue can be sent to **info@aiagentsservices.net** or via the form at **[ars-1.org/#feedback](https://ars-1.org/#feedback)**.

## Authorised issuer programme

If you represent a body that may be interested in becoming an authorised issuer of AIS-1 identities — particularly under the Sovereign tier, for use in the State-to-Citizen profile — please reach out via email.

## Standards-body engagement

ARS-1 is on track for submission to ISO TC 68 / SC 8 (Financial Services) during the Phase 1.0 standardisation track (Q2 2027). Earlier engagement from ISO 20022 registration authorities, SWIFT, the FATF, the BIS Innovation Hub, and the FSB is particularly welcome.

## Licensing of contributions

By contributing, you agree that your contribution is released under the same CC0 1.0 Universal license that covers the rest of the repository. This means your contribution is dedicated to the public domain — anyone may use it for any purpose without restriction.

## Code of conduct

Be respectful. Disagreements about technical substance are welcome; personal attacks are not. The ARS-1 team reserves the right to close issues or PRs that don't engage with the technical substance.
